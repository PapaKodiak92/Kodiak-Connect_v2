# Kodiak Connect v1.1.5 Dev Update

We pushed another Kodiak Connect stabilization pass focused on mobile usability, media reliability, and the packaged app release path.

## Fixed

- Fixed Matrix media previews and downloads that could fail for other users with `M_MISSING_TOKEN`.
- Confirmed media sent from one account can now be viewed from another account after signing in separately.
- Added a mobile workspace layout repair pass so the app is more usable on smaller screens.
- Improved mobile access to the channel strip, chat view, message composer, media panel, profile/settings modals, and safety/report dialogs.

## Deployment Notes

- Confirmed the live web deployment path on the VPS.
- The web app is served from `/var/www/kodiak-connect`, so frontend builds must be synced there after `npm run build`.
- Confirmed packaged Windows, Linux, and Android apps use bundled frontend assets, so they require a versioned release instead of only a live web patch.

## Release Pipeline

- Started the `1.1.5` packaged app release using the existing `release-all.ps1` workflow.
- Built the Windows MSI and updater signature.
- Built the Android debug APK.
- Added the missing local Tauri CLI dependency needed for release builds.
- Found the VPS Linux build blocker caused by repo ownership/permissions under `/opt/kodiak-connect`.

## Next

- Finish the Linux DEB publish step after VPS repo permissions are corrected.
- Publish the final Windows, Linux, and Android update artifacts.
- Verify the desktop and Android clients detect the new update manifests.
