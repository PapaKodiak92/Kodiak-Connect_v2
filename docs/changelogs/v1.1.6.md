# Kodiak Connect v1.1.6 Dev Update

Kodiak Connect v1.1.6 is a hotfix/stability update focused on restoring Matrix stability, fixing media behavior, and cleaning up the packaged app release path.

## Fixed

- Restored Matrix/Synapse service stability after the VPS runtime permission issue.
- Fixed Postgres data ownership for the Matrix database volume.
- Fixed Synapse data ownership so the homeserver could start normally again.
- Confirmed Matrix registration/admin writes work again by creating a rescue admin account successfully.
- Restored account recovery ability for the main admin account.
- Fixed authenticated Matrix media rendering for uploaded images, screenshots, audio, video, and files.
- Restored GIF preview behavior so Giphy shares render correctly instead of falling back to raw text.
- Added a safer profile avatar fallback when Matrix media upload is unavailable.
- Improved mobile workspace layout for smaller screens.

## Release Pipeline

- Confirmed packaged apps need versioned releases because Windows, Linux, and Android builds use bundled frontend assets.
- Continued using the existing elease-all.ps1 release workflow.
- Fixed the local Tauri CLI build blocker.
- Fixed the VPS Linux build blocker caused by repo ownership permissions.
- Identified and corrected the dangerous broad ownership change that affected runtime database volumes under /opt/kodiak-connect/runtime.

## Deployment Notes

- The live web app is served from /var/www/kodiak-connect.
- Packaged update artifacts are served from updates.kodiak-connect.com.
- Matrix runtime data lives under /opt/kodiak-connect/runtime/matrix.
- Future permission fixes must keep repo files, web files, Synapse data, and Postgres data owned separately.

## Next

- Continue testing the v1.1.6 app packages on Windows, Linux, and Android.
- Keep validating media upload/download behavior with multiple accounts.
- Add a safer VPS permission repair script so runtime database volumes are never chowned as repo files again.
